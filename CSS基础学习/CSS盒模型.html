📦 CSS 盒模型 · 元素分解

content · padding · border · margin · 每一部分单独可视

🧱 标准盒模型 (content-box)

width:200px; padding:20px; border:5px;
.box-demo { width: 200px; padding: 20px; border: 5px solid #2563eb; margin: 20px; background: #dbeafe; }
总宽 = 200+40+10 = 250px

🔁 怪异盒模型 (border-box)

box-sizing: border-box;
.box-border-box { width: 200px; padding: 20px; border: 5px solid #eab308; box-sizing: border-box; }
总宽200px (内容压缩)

🔍 拆解 · 内容 / 内边距 / 边框 / 外边距

📄 内容区域 · content-only
.content-only { width: 200px; height: 80px; background: #7dd3fc; /* 无 padding, border, margin */ }
content 200x80

只定义内容区，无内边距/边框/外边距

🧽 内边距 · padding-only
.padding-only { width: 200px; padding: 20px; background: #b7c9fe; /* border:0; margin:0; content-box */ }
padding 20px 四周
内容宽仍为200px

总宽度 = 200 + 40 = 240px (背景延伸至padding)

🖼️ 边框 · border-only
.border-only { width: 200px; border: 10px solid #f97316; background: #fed7aa; line-height: 60px; height: 60px; }
边框10px 内容200x60

总宽 = 200 + 20 = 220px，边框外扩

↔️ 外边距 · margin-only
.margin-parent { background:#acb8c7; overflow:hidden; }
.margin-child { margin:30px; height:60px; background:#bbf7d0; }
margin 30px

灰色区域为父背景，显露透明margin。元素本身无padding/border

⚡ 以上示例均采用默认 box-sizing: content-box (边框和内边距向外扩展)。怪异盒模型效果请参考上一节。

📏 外边距折叠 (Margin Collapse)

上下相邻margin合并
.margin-collapse-demo { margin: 20px 0; }
第一个div margin=20
第二个div margin=20，实际间距20而非40

📐 盒模型总尺寸计算公式

总宽度 = width + padding-left + padding-right + border-left + border-right
总高度 = height + padding-top + padding-bottom + border-top + border-bottom

👉 若使用 box-sizing: border-box; 则 width 包含 padding 和 border
内容区宽度会被压缩：内容宽度 = width − 左右padding − 左右border

⭐ 外边距(margin)不影响元素实际占位尺寸，但影响布局间距，且可能发生折叠。

📏 视口单位 (vw / vh)

🌐 宽度 50vw (视口宽度的50%)
.vw-demo { width: 50vw; height: 80px; background: #a5f3fc; line-height: 80px; text-align: center; font-weight: 500; }
宽度 = 50vw

调整浏览器窗口水平大小，观察蓝色条宽度变化。

📐 高度 30vh (视口高度的30%)
.vh-demo { height: 30vh; width: 200px; background: #bef264; line-height: 30vh; /* 垂直居中 */ text-align: center; font-weight: 500; }
高度 = 30vh

调整浏览器窗口垂直大小，观察绿色块高度变化。

💡 vw 和 vh 是相对于视口宽度和高度的百分比单位。
• 1vw = 视口宽度的1%   • 1vh = 视口高度的1%
利用它们可以创建随视口缩放的响应式元素。
⚡ 盒模型是CSS布局的基石 —— 理解 content、padding、border、margin 各自的作用，才能精准控制元素尺寸与间距。
