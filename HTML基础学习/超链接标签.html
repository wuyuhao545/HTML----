🧩 综合示例

这是一个段落，包含 普通链接， 一个 内部锚点， 一个 📧 邮件链接， 和一个 按钮链接。

🧭 导航菜单与下拉菜单（今日所学）

导航菜单的可点击性 必须使用 <a> 
<ul>
  <li><a href="#">首页</a></li>
  <li><a href="#">关于我们</a></li>
  <li><a href="#">服务</a></li>
  <li><a href="#">联系</a></li>
</ul>
  • 首页
  • 关于我们
  • 服务
  • 联系

即使暂时用 href="#" 占位，也必须用 <a> 包裹，确保可点击、键盘可访问、语义正确。

CSS :hover 下拉菜单 二级菜单示例

}

  <li class="nav-item">
    <a href="#">产品</a>
    <ul class="dropdown">
      <li><a href="#">手机</a></li>
      <li><a href="#">电脑</a></li>
    </ul>
  </li>
</ul>
  • 产品

    • 电脑

  • 服务

⚠️ 局限： 纯CSS :hover 下拉菜单在移动端无法通过点击展开，键盘用户也无法通过 Tab 展开子菜单。实际项目通常需要 JavaScript 增强可访问性。

⚠️ 常见错误：未闭合的 <a> 标签

<a href="...">链接文字 缺少 </a> 
<li><a href="brief.html">医院简介</li>

上面的代码中，<a> 标签没有闭合，浏览器解析时会将后续内容（如“组织架构”）误认为是 <a> 的一部分，导致布局异常、样式错乱，甚至链接失效。

后果模拟：

医院简介（本应为独立项，但由于未闭合，后续内容也被包含）
组织架构（实际会被包裹在上一个链接内，导致可点击区域异常）

实际渲染中，第二个列表项会变成链接的一部分，点击第一个区域可能跳转，第二个区域样式混乱。

✅ 正确写法： <li><a href="brief.html">医院简介</a></li>
如何避免
  • 使用代码编辑器的自动补全功能，避免手动输入标签。
  • 养成写完开始标签立即写结束标签的习惯。
  • 使用HTML验证工具（如W3C验证器）检查页面。
  • 在浏览器开发者工具中检查元素，查看是否有意外的嵌套关系。
