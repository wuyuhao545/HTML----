🎧 监听方式 内联 · DOM0 · DOM2
内联事件 (HTML)
直接在标签中使用 onclick 属性，this 指向 window，可传参。
<button onclick="handler(this)">点击</button>
点击弹出提示，注意 this 指向 window，但传参可获取元素。
DOM0 (属性赋值)
通过 element.onclick = fn 绑定，后覆盖前。
btn.onclick = function() { alert('第一次'); };
btn.onclick = function() { alert('第二次'); };
只会弹出“第二次”，说明被覆盖。
DOM2 addEventListener
可绑定多个监听器，顺序执行，支持捕获/冒泡控制。
btn.addEventListener('click', fn1);
btn.addEventListener('click', fn2);
依次弹出 1 和 2，共存不覆盖。
📦 事件对象 常用属性与方法
target / currentTarget
target 是实际触发事件的元素，currentTarget 是绑定监听器的元素。
parent.addEventListener('click', (e) => {
    console.log('target:', e.target.tagName);
    console.log('currentTarget:', e.currentTarget.tagName);
});
父元素 (点击此处或按钮)

点击按钮或父元素空白区域，观察 target 与 currentTarget 的区别。
preventDefault()
阻止默认行为，如链接跳转、表单提交。
link.addEventListener('click', (e) => {
    e.preventDefault();
    console.log('链接被阻止');
});
点击此链接 (默认跳转被阻止)
点击链接不会跳转，弹出提示。
stopPropagation()
阻止事件冒泡（或捕获）进一步传播。
child.addEventListener('click', (e) => {
    e.stopPropagation();
    console.log('子元素点击');
});
父元素
点击子按钮只弹出子元素消息，父元素监听器未触发。
clientX / clientY · 修饰键
鼠标事件中获取光标位置，并检测 Ctrl / Shift / Alt 是否按下。
element.addEventListener('click', (e) => {
    console.log(`X: ${e.clientX}, Y: ${e.clientY}`);
    console.log(`Ctrl: ${e.ctrlKey}, Shift: ${e.shiftKey}`);
});

点击后显示结果

点击区域，下方显示鼠标坐标及当时按下的功能键（可配合 Ctrl/Shift/Alt 测试）。
button / buttons (鼠标按键)
button 表示按下的具体按钮（0左键，1中键，2右键）；buttons 表示同时按下的按钮组合（位掩码）。
element.addEventListener('mousedown', (e) => {
    console.log('button:', e.button);
    console.log('buttons:', e.buttons);
});
在此区域按下鼠标左/中/右键

在不同按键按下时，观察 button 值和 buttons 掩码的变化。
key / code (键盘事件)
key 表示输入的字符（考虑Shift等），code 表示物理按键代码。
input.addEventListener('keydown', (e) => {
    console.log('key:', e.key, 'code:', e.code);
});

在输入框中按键，下方显示 key（输入的字符）和 code（物理按键）。
eventPhase (事件阶段)
表示事件当前处于哪个阶段：1捕获，2目标，3冒泡。
element.addEventListener('click', (e) => {
    console.log('phase:', e.eventPhase);
}, true); // 捕获
父元素

点击按钮，观察事件流在不同阶段（父捕获、目标、父冒泡）显示的事件阶段值。
timeStamp · defaultPrevented
timeStamp 事件创建的时间戳（毫秒），defaultPrevented 表示是否调用过 preventDefault。
element.addEventListener('click', (e) => {
    console.log('timeStamp:', e.timeStamp);
    console.log('defaultPrevented:', e.defaultPrevented);
});
点击链接 (阻止跳转)

点击链接，显示事件时间戳和 preventDefault 调用状态。
🌊 事件流 捕获与冒泡顺序
addEventListener 的第三个参数
true = 捕获阶段，false/省略 = 冒泡阶段。
parent.addEventListener('click', fn, true);  // 捕获
parent.addEventListener('click', fn, false); // 冒泡
outer (外层)
inner (内层) —— 点击我
点击内层，弹出顺序：outer 捕获 → inner 捕获 → inner 冒泡 → outer 冒泡。
🤝 事件委托 利用冒泡处理动态元素
委托模式
将监听器加到父元素，通过 target 判断子元素，动态添加的元素也能响应。
parent.addEventListener('click', (e) => {
    if (e.target.matches('li')) {
        console.log('点击了', e.target.textContent);
    }
});

🧹 移除事件 removeEventListener
必须传入相同函数引用
匿名函数无法移除，需使用具名函数。
const handler = () => {};
btn.addEventListener('click', handler);
btn.removeEventListener('click', handler);
先点击上方按钮弹出提示；点击“移除”后，再点击上方按钮不再弹出。
总结对比
  • ✅ DOM0：简单但后覆盖前
  • ✅ DOM2：多监听、事件流控制
  • ✅ 事件对象：提供丰富信息
  • ✅ 事件委托：减少监听器、动态友好

推荐使用 addEventListener 进行事件处理。

⚠️ 错误处理 try/catch · throw · 调试
try/catch 捕获异常
将可能出错的代码放在 try 块，catch 中处理错误。
try {
    throw new Error('演示错误');
} catch(e) {
    console.log(e.message);
}
点击按钮，try 中抛出错误，catch 捕获后弹出提示。
throw 自定义错误
手动抛出错误（可以是任何类型，通常用 Error 对象）。
if(x < 0) {
    throw new Error('x不能为负数');
}
模拟检查负数，小于0则抛出错误并被捕获。
调试技巧 (console)
使用 console.log、console.error、console.table 等输出信息。
console.log('普通日志');
console.error('错误样式');
console.table([{a:1}]);
点击后打开浏览器开发者工具（Console）查看不同级别的输出。
