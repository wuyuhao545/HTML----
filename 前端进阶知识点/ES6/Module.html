📤 命名导出 · import 指定

命名导出 (Named Export)基础用法
// module: math.js
export const PI = 3.14159;
export function square(x) { return x * x; }
export class Calculator { ... }

// import
import { PI, square } from './math.js';
📌 导出多个成员，导入时需使用相同的名称（可重命名）。
重命名导入as 关键字
import { PI as piValue, square as sq } from './math.js';
console.log(piValue, sq(5));
✏️ 使用 `as` 避免命名冲突或简化名称。
命名空间导入* as
import * as MathLib from './math.js';
console.log(MathLib.PI);
console.log(MathLib.square(4));
🌐 将模块所有导出挂载到一个对象上，避免重复导入。

🎯 默认导出 · 混合导出

默认导出 (Default Export)每个模块仅一个
// module: logger.js
export default function log(msg) {
  console.log(`[LOG] ${msg}`);
}
// import
import log from './logger.js';
📢 默认导出可使用任意名称导入，无需花括号。
混合导出命名 + 默认
// module: utils.js
export const version = '1.0';
export default function init() { ... }
// import
import init, { version } from './utils.js';
🔀 一个模块可以同时包含默认导出和命名导出。
重新导出 (Re-export)聚合模块
// index.js
export { PI, square } from './math.js';
export { default as log } from './logger.js';
📚 将其他模块的导出集中转发，便于统一入口。

⚡ 动态导入 · 模块作用域 · 加载顺序

动态导入 (Dynamic Import)按需加载
import('./module.js').then(module => {
  module.doSomething();
});
// 或 await import('./module.js')
🚀 运行时异步加载，用于代码分割、懒加载。
模块作用域变量隔离
// 模块内部声明的变量不会污染全局
const secret = "模块私有";
export const api = "公开";
🛡️ 每个模块拥有独立作用域，顶层 `this` 为 undefined。
模块加载顺序深度优先
// a.js → b.js → c.js
// 执行顺序按照依赖关系深度优先，静态分析确保无循环依赖
📈 ES Module 在编译时确定依赖，保证执行顺序可预测。
💡 ES Module 是 JavaScript 官方标准化模块系统，支持静态分析、Tree Shaking，现代开发必备。
