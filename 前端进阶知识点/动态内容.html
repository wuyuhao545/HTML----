🎯 动态内容 · 选择器适配

当 JavaScript 动态加载 HTML 后，DOM 结构可能变化，原有 CSS 选择器需要同步调整
错误选择器：.main > ul（只匹配直接子元素）
正确选择器：.main #nav-container ul（匹配新结构）
❌ 未适配选择器 样式失效

原本使用 .main > ul，但动态加载后 ul 外面包了一层 #nav-container，导致样式丢失。

/* 错误：仍使用子选择器 */ .main > ul { background: #fecaca; padding: 1rem 2rem; border-radius: 50px; list-style: none; display: flex; gap: 1.5rem; font-weight: 500; border: 2px solid #ef4444; }
等待动态加载…

加载后观察：红色边框+浅红背景 应为 .main > ul 的效果，但由于 ul 被 div 包裹，样式不生效（背景为白色）。

✅ 已适配选择器 样式生效

调整选择器为 .main #nav-container ul，精确匹配新结构。

/* 正确：适配新结构 */ .main #nav-container ul { background: #bbf7d0; padding: 1rem 2rem; border-radius: 50px; list-style: none; display: flex; gap: 1.5rem; font-weight: 500; border: 2px solid #22c55e; }
等待动态加载…

加载后观察：绿色边框+浅绿背景 正确应用，因为选择器匹配了 #nav-container 内的 ul。

📌 核心原则： 当使用 JavaScript 动态插入 HTML 片段（如 fetch 加载公共组件）时，DOM 结构可能从 .main > ul 变为 .main > div > ul 或 .main > 某个容器 > ul。
此时如果 CSS 仍使用旧的选择器（尤其是子选择器 > 或依赖特定层级的选择器），样式就会丢失。
解决方法：审查最终生成的 DOM 树，然后调整选择器以匹配实际结构，例如从 .main > ul 改为 .main .nav-container ul 或 .main #nav-container ul。
⚡ 动态内容加载后，请使用浏览器开发者工具检查实际 DOM 结构，并相应更新 CSS 选择器。
