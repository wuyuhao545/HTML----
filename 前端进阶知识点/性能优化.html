📦 CSS合并 减少HTTP请求

多个CSS文件的问题
<link rel="stylesheet" href="reset.css"> <link rel="stylesheet" href="typography.css"> <link rel="stylesheet" href="layout.css"> <link rel="stylesheet" href="theme.css">
reset.css typography.css layout.css theme.css

4个CSS文件 = 4次HTTP请求，增加页面加载时间，尤其在移动网络下。

合并成一个文件
<link rel="stylesheet" href="style.css">
style.css (合并后)

1次HTTP请求，减少延迟，利用HTTP/2的 multiplexing 效果更好，但合并仍有意义。

1请求 vs 之前 4请求
自动化合并（构建工具）
// webpack / vite 配置 import './reset.css'; import './typography.css'; import './layout.css'; import './theme.css';

现代前端工具（Webpack、Vite、Gulp）会自动将多个CSS合并成一个文件，并压缩优化。

也可手动合并，但推荐使用构建工具自动化处理。

🖼️ 图片优化 尺寸适当 · 避免变形

拉伸变形的常见原因
<img src="photo.jpg" width="200" height="100"> <!-- 但原图可能是 400x400，强制拉伸 -->
原图 (400x400)
400x400
强制拉伸显示 (200x100)
变形了

设置固定宽高但比例与原图不符导致变形。

保持宽高比
<img src="photo.jpg" style="width:200px; height:auto;"> <!-- 或设置 max-width: 100%; height: auto; -->
高度自动，比例不变
200x200 (等比例)

只设置宽度，高度自适应，或使用 object-fit 控制。

响应式图片 (srcset)
<img srcset="small.jpg 400w, medium.jpg 800w, large.jpg 1200w" sizes="(max-width: 600px) 400px, (max-width: 1200px) 800px, 1200px" src="fallback.jpg">

根据屏幕宽度加载不同尺寸图片，避免移动端加载超大图。

📱 小屏加载 small.jpg (400w)
💻 大屏加载 large.jpg (1200w)
object-fit 解决拉伸
.img-container { width: 200px; height: 100px; } .img-container img { width: 100%; height: 100%; object-fit: cover; /* 或 contain */ }
cover
cover (裁剪)
contain
contain (留白)

object-fit 可控制图片在固定容器内的显示方式，避免变形。

格式选择与压缩
- 照片 → JPEG / WebP - 图标/插画 → PNG / SVG - 动图 → GIF / WebP - 压缩工具：TinyPNG, ImageOptim

选择合适的格式可大幅减小体积。例如使用 WebP 替代 JPEG 可减少 25-35% 大小。

⏱️ 防抖与节流 控制高频事件，提升性能

防抖 (Debounce)
将多次高频操作合并为一次，只在最后一次操作后的延迟时间后执行。适合输入框搜索、窗口resize结束等场景。
function debounce(fn, delay) { let timer; return function(...args) { clearTimeout(timer); timer = setTimeout(() => fn.apply(this, args), delay); }; } // 使用 input.addEventListener('input', debounce(function(e) { console.log('搜索:', e.target.value); }, 500));

📝 连续输入时，只在停止输入500ms后触发一次搜索请求。

输入 "hello" 快速敲击 → 只触发一次搜索
节流 (Throttle)
保证一段时间内只执行一次，稀释执行频率。适合滚动事件、鼠标移动、频繁点击等。
function throttle(fn, interval) { let lastTime = 0; return function(...args) { const now = Date.now(); if (now - lastTime >= interval) { fn.apply(this, args); lastTime = now; } }; } // 使用 window.addEventListener('scroll', throttle(function() { console.log('滚动位置:', window.scrollY); }, 200));

🖱️ 滚动时，每200ms最多触发一次，避免频繁计算。

快速滚动 → 控制台打印频率降低
对比与选择
防抖只执行最后一次输入、调整窗口
节流均匀稀释执行滚动、鼠标移动

⚠️ 两者都用于性能优化，防止高频回调阻塞主线程。

Lodash 提供了现成的 _.debounce 和 _.throttle，可直接使用。

✅ 推荐在实际项目中引入 lodash 或自己实现简单的工具函数。

正确优化 vs 未优化

❌ 未优化
