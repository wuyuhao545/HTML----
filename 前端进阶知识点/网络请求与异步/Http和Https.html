📖 协议基础

HTTP 请求方法RESTful 核心
GET → 获取资源
POST → 创建资源
PUT → 完整更新
PATCH → 部分更新
DELETE → 删除资源
🔹 每种方法对应不同的语义，遵循幂等性设计。
常见状态码1xx-5xx
200 OK
201 Created
301 Moved Permanently
400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found
500 Internal Server Error
🔢 状态码表明请求结果，分为信息、成功、重定向、客户端错误、服务器错误。
HTTP 报文结构请求/响应
请求行: POST /api/users HTTP/1.1
请求头: Host: example.com
请求体: { "name": "Alice" }
---
状态行: HTTP/1.1 200 OK
响应头: Content-Type: application/json
响应体: { "id": 1 }
📨 报文包含起始行、头部字段、空行和主体。

🔒 HTTPS · 安全传输

HTTPS 加密流程SSL/TLS
1. 客户端发起 HTTPS 请求
2. 服务器返回数字证书
3. 客户端验证证书
4. 生成随机对称密钥
5. 用公钥加密对称密钥发送给服务器
6. 服务器用私钥解密，之后使用对称加密通信
🔐 通过混合加密（非对称+对称）保证数据机密性和完整性。
HTTP vs HTTPS安全性对比
HTTP : 明文传输，易被窃听/篡改
HTTPS: 加密传输，防窃听、防篡改、防冒充
⚡ 现代网站全面推荐使用 HTTPS，提升安全性和用户信任。
证书与CA数字证书
证书包含：域名、公钥、颁发机构、有效期等
由 CA 签发，用于验证服务器身份
📜 浏览器内置受信任 CA 根证书，确保通信方身份真实。
💡 HTTP 协议是 Web 的基石，HTTPS 为其提供了安全保障，深入理解对开发调试至关重要。
