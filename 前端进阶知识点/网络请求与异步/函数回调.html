📞 回调函数异步处理 · 传统模式

回调地狱 · 定时器 · 基于回调的异步请求

🔄 回调基础演示

setTimeout 回调异步延迟
console.log("开始");
setTimeout(() => {
  console.log("2秒后执行");
}, 2000);
console.log("结束");
⏰ 异步回调不阻塞主线程，延迟执行。
回调地狱示例嵌套回调
setTimeout(() => {
  console.log("第1步");
  setTimeout(() => {
    console.log("第2步");
    setTimeout(() => {
      console.log("第3步");
    }, 500);
  }, 500);
}, 500);
📦 多层嵌套导致代码难以维护，被称为“回调地狱”。
回调式异步请求模拟XHR回调
function fetchData(callback) {
  setTimeout(() => {
    callback({ id: 1, name: "回调数据" });
  }, 1000);
}
fetchData(data => console.log(data));
📨 将异步结果通过回调函数返回。
💡 回调函数是最早的异步解决方案，但易产生“回调地狱”，后续被Promise和async/await取代。
